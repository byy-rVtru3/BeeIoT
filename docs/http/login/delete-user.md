# Удаление аккаунта пользователя

## Описание
Эндпоинт для полного удаления аккаунта пользователя из системы. Удаляет пользователя из базы данных и все связанные с ним JWT токены. Требует авторизации.

## HTTP запрос
```
DELETE /api/auth/delete/user
```

## Заголовки
```
Authorization: Bearer {jwt_token}
```

## Тело запроса
```
```

## Ответы

### 200 OK
Аккаунт успешно удален.
```json
{
  "status": "ok",
  "message": "Ваш аккаунт успешно удален",
  "data": null
}
```

### 401 Unauthorized
Токен не действителен или отсутствует авторизация.
```json
{
  "status": "error",
  "message": "Unauthorized"
}
```

### 500 Internal Server Error
Внутренняя ошибка сервера.
```json
{
  "status": "error",
  "message": "Внутренняя ошибка сервера"
}
```

## Безопасность
- Эндпоинт защищен middleware авторизации
- Требует валидный JWT токен в заголовке Authorization
- Email пользователя автоматически извлекается из контекста авторизации
- Операция необратима - восстановить удаленный аккаунт невозможно

## Последствия удаления
1. **База данных**: Пользователь полностью удаляется из PostgreSQL
2. **Сессии**: Все JWT токены пользователя удаляются из Redis
3. **Данные**: Все связанные с пользователем данные удаляются
4. **Безопасность**: Все активные сессии немедленно становятся недействительными

## ⚠️ Предупреждение
Удаление аккаунта является **необратимой операцией**. После удаления:
- Восстановить данные будет невозможно
- Потребуется новая регистрация для создания аккаунта
- Все связанные данные будут потеряны
