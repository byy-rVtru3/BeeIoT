# Выход из системы

## Описание
Эндпоинт для выхода пользователя из системы. Удаляет JWT токен из активных сессий. Требует авторизации.

## HTTP запрос
```
DELETE /api/auth/logout
```

## Заголовки
```
Authorization: Bearer {jwt_token}
```

## Тело запроса
Тело запроса отсутствует. JWT токен передается в заголовке авторизации.
```

## Тело запроса
```

## Тело запроса
```json
{
  "jwt": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

## Ответы

### 200 OK
Выход из системы выполнен успешно.
```json
{
  "status": "ok",
  "message": "Вы успешно вышли из системы",
  "data": null
}
```

### 400 Bad Request
Неверный формат JSON.
```json
{
  "status": "error",
  "message": "Неверный формат JSON"
}
```

### 401 Unauthorized
Токен не действителен или отсутствует авторизация.
```json
{
  "status": "error",
  "message": "Unauthorized"
}
```

### 500 Internal Server Error
Внутренняя ошибка сервера.
```json
{
  "status": "error",
  "message": "Внутренняя ошибка сервера"
}
```

## Безопасность
- Эндпоинт защищен middleware авторизации
- Требует валидный JWT токен в заголовке Authorization
- После выполнения указанный токен становится недействительным
- Email пользователя извлекается из контекста авторизации
